# Scale Stage

## Goal
Build systematic growth sustained by mature operations. Create a defensible moat through accumulated product depth, integration depth, and proprietary data/workflows.

## Exit Criteria
Company is sustainable even as founder is increasingly not running day-to-day. Typically takes one of three forms:
- Sustainable profitability without external capital
- IPO-readiness
- Acquisition readiness

## Challenges & Antidotes

| Challenge | Antidote |
|-----------|----------|
| **Delegating the operational layer** | Codify institutional knowledge into documented, auditable, transferable systems |
| **Scaling technical operations** | Build docs, support playbooks, SLAs, logging, monitoring, incident response |
| **Scaling organizational functions** | Treat hiring, payroll, accounting, legal as infrastructure; build or automate systematically |
| **Building a GTM function** | AI bootstraps GTM: segmentation, messaging, sales playbooks, content pipelines |

## Key Exercises

### 1. Bottleneck map
Inventory every workflow, decision, and approval routed through you. Ask: "What happens when I'm unavailable for a week?"

Workflows that stall need:
- Handoff criteria
- Escalation paths
- Exception handling

### 2. Enterprise gap analysis
- Identify 3 ideal enterprise customers
- Ask Claude: what documentation, SLAs, and support infrastructure would their procurement expect?
- Sequence technical and documentation work across Claude Code and Cowork

### 3. Build a real GTM function
**Chat:**
- Market segmentation and messaging architecture
- Analyst relations strategy
- Sales playbooks
- Investor-facing metrics narratives

**Claude Cowork:**
- Content pipelines
- Outbound sequences
- CRM hygiene
- Pipeline reporting
- PR cadences

**Claude Code:**
- Interactive demo environments
- Integration documentation
- Sandbox tenants
- API references
- Technical one-pagers

### 4. Domain expertise → moat
- Capture industry jargon, regulatory gotchas, edge cases into structured context
- Codify recurring workflows into reusable skills
- Build dedicated test cases for edge cases a generic competitor would get wrong
- Your test suite becomes a map of your moat

### 5. Data flywheel narrative
- Identify 3 highest-signal behavioral patterns in user interaction data
- Design feedback loops that turn each into systematic model improvement
- Draft a one-page moat narrative: how your data flywheel works, how long it's been spinning, and why a competitor couldn't replicate it in under 2 years

### 6. Workflow lock-in audit
- Map top 10 customers by integration depth: automations, integrations, team workflows, switching cost
- Build native integrations, APIs, webhooks, SDKs so customers build **on top of** your product
- Identify which integration types create deepest lock-in for your specific product
